# NextAuth

**Auth for Next.js**

Authentication for Next.js and Serverless. Supports OAuth, email, credentials, and custom providers with a simple API.

## Key Features

- OAuth & credentials
- Next.js native
- Open source

## Setup

<Steps>
  <Step title="Install NextAuth">
    <CodeGroup>
    ```bash npm
    npm install next-auth
    ```
    
    ```bash yarn
    yarn add next-auth
    ```
    
    ```bash pnpm
    pnpm add next-auth
    ```
    </CodeGroup>
  </Step>
  
  <Step title="Create API route">
    Create `app/api/auth/[...nextauth]/route.ts` and configure providers (Google, GitHub, etc.).
  </Step>
  
  <Step title="Add environment variables">
    Add NEXTAUTH_URL and provider client IDs/secrets to `.env`.
  </Step>
  
  <Step title="Wrap your app">
    Wrap the app in SessionProvider.
  </Step>
</Steps>

## Usage

<Accordion title="Server components">
  Use `getServerSession()` in server components to access the current session.
</Accordion>

<Accordion title="Client components">
  Use `useSession()` in client components to access the current session.
</Accordion>

<Accordion title="Protect API routes">
  Protect API routes by checking the session; redirect unauthenticated users as needed.
</Accordion>

## Hackathon tips

- OAuth providers are quick to add; no custom UI required.
- Open source and self-hosted—no vendor lock-in, great for open-source projects.
